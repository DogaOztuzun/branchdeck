## Vision

### 10x Check
Branchdeck daemon as a standardized agent execution platform with OpenAPI spec, SDK clients in Python/TypeScript, a plugin system for custom orchestration flows, and a marketplace where people share orchestration templates. Think "Docker Hub for agent workflows."

### Strategic Framing
The external feedback says: "You're not building another Cursor wrapper. You're building the dev operating system for the multi-repo, agent-augmented future." The daemon extraction is the inflection point from "desktop tool" to "platform with API."

Target audience: AI/agent builders who use Claude Code. The curl demo IS the product demo for this audience. First demo: `curl localhost:13371/api/repos`.

### Premise Validation
- Demo-first (Approach D) was rejected: building multi-run inside Tauri's single-process model means rewriting the concurrency model during extraction. More expensive than doing it right the first time.
- Parallel work (Approach E) was rejected: solo developer on interconnected systems creates merge conflicts. Sequential is better.
- Daemon-first (Approach A) confirmed: extraction is mechanical (75% of files move unchanged), and the API is the product for this audience.

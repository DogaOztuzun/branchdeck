## Context
A brand new user opens Branchdeck for the first time. The workspace is empty — no repos have been added yet. They want to add their first git repository to start working.

## Steps
1. Launch Branchdeck and observe the initial empty state
2. Look for a way to add a repository (button, menu, or prompt)
3. Select or enter the path to a local git repository
4. Confirm the repository is added and appears in the left sidebar
5. Verify the repo name, current branch, and any worktrees are displayed

## Expected Satisfaction
- The empty state should clearly guide the user toward adding a repo
- Adding a repo should feel instant — no unexplained loading or delay
- The newbie should understand what a "repository" means in context without jargon

## Edge Cases
- The user selects a directory that is not a git repository
- The user selects a bare repository with no working directory
- The path contains spaces or special characters
